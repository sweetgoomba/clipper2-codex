# Template Builder Role-Scoped Common Styles Design

## 목적

Template Builder에서 텍스트 스타일과 텍스트 효과는 비율별 개별 값이 아니라, 같은 역할을 가진 레이어들에 대해 비율 공통으로 유지되는 기본값이어야 한다.

예를 들면:

- `메인 타이틀`의 폰트, 색상, 크기, 자간, 행간, 박스, 윤곽선, 그림자는 모든 비율에 공통 적용된다.
- `서브 타이틀`은 메인 타이틀과 별도의 공통 스타일을 가진다.
- `자막`도 별도의 공통 스타일을 가진다.
- 비율별로 달라지는 것은 주로 위치, 크기, 줄 모드, 표시 여부, 배치 값이다.

현재 문제는 이런 역할별 스타일이 각 ratio variant 안에 흩어져 있어서, 한 비율에서 수정하면 다른 비율에 자연스럽게 전파되지 않는다는 점이다.

## 설계 원칙

- `공통 레이어`는 레이아웃 전용으로 유지한다.
- 역할별 공통 스타일은 왼쪽 목록에 별도 섹션으로 두지 않는다.
- 공통성은 오른쪽 인스펙터에서만 드러낸다.
- 사용자는 레이어를 선택한 뒤, 해당 역할의 `비율 공통 스타일`을 수정한다는 것을 명확히 인지해야 한다.
- 비율별 차이는 위치/크기/배치 쪽에 남기고, 스타일은 역할 단위로 공유한다.

## 데이터 모델

### 가족 단위 공통 스타일

가족(`TemplateBuilderFamily`)에 역할별 공통 스타일 저장소를 둔다.

```ts
family.commonStyles = {
  mainTitle: { text, box, outline, shadow },
  subTitle: { text, box, outline, shadow },
  bottomTitle: { text, box, outline, shadow },
  subtitleText: { text, box, outline, shadow },
  logoText: { text, box, outline, shadow },
};
```

### 역할 단위 스타일 범위

- `mainTitleLine1` / `mainTitleLine2`는 같은 `mainTitle` 공통 스타일을 공유한다.
- `subTitle`는 `subTitle` 공통 스타일을 공유한다.
- `bottomTitle`는 `bottomTitle` 공통 스타일을 공유한다.
- `subtitleText`는 `subtitleText` 공통 스타일을 공유한다.
- `logoText`는 `logoText` 공통 스타일을 공유한다.
- `layoutImage`와 `logoImage`는 공통 타이포그래피 대상이 아니다.

### 비율별 값

각 variant는 다음만 주로 보유한다.

- `x`, `y`, `width`, `height`
- `lineMode`
- `oneLineY`, `twoLineFirstY`, `twoLineSecondY`
- `visible`
- 필요 시 역할별 override

### 우선순위

렌더와 편집 시 effective style은 다음 순서로 해석한다.

1. variant override
2. family role common style
3. 시스템 기본값

## UI 구조

### 왼쪽 패널

- `공통 레이어`
  - `레이아웃`만 포함
- `기본 레이어`
  - `메인 타이틀`
  - `서브 타이틀`
  - `하단 타이틀`
  - `자막`
  - `로고 텍스트`

### 오른쪽 인스펙터

기본 레이어를 선택했을 때 인스펙터에는 다음이 보인다.

- `위치와 크기`
  - 비율별로 다른 값
- `비율 공통 스타일`
  - 짧은 설명 문구: `선택한 역할의 모든 비율에 적용됩니다.`
  - `텍스트 스타일`
  - `텍스트 효과`

`텍스트 스타일`과 `텍스트 효과`는 별도 의미의 공통 섹션이 아니라, 같은 역할의 비율 공통 기본값을 편집하는 한 묶음으로 취급한다.

## 편집 동작

- 사용자가 `16:9`에서 `메인 타이틀`의 폰트를 바꾸면, 같은 역할의 다른 비율에도 반영된다.
- 사용자가 `자막`의 색상을 바꾸면, `자막` 역할의 모든 비율에 반영된다.
- `mainTitleLine1`과 `mainTitleLine2`는 같은 공통 스타일을 공유하므로, 줄 위치만 다르고 스타일은 같이 움직인다.
- `공통 레이어`에 있는 레이아웃은 기존처럼 비율별 개별 조정 가능하다.

## 마이그레이션

기존 데이터는 가족별 역할 공통 스타일이 없을 수 있다.

- 초기 로드 시, 우선순위 ratio(`16:9` -> `4:3` -> `1:1` -> `full`)에서 스타일을 읽어 공통 스타일의 초기값을 만든다.
- 이후 편집 시에는 공통 스타일 저장소를 우선 사용한다.
- 기존 ratio별 스타일 값은 즉시 삭제하지 않고, 호환 기간 동안 fallback으로 유지할 수 있다.

## 수용 기준

- `공통 레이어`에는 레이아웃만 보인다.
- 기본 레이어 선택 시 인스펙터에 `비율 공통 스타일`이 표시된다.
- 텍스트 스타일/효과를 수정하면 같은 역할의 모든 비율에 반영된다.
- 비율 전환 후에도 스타일은 유지되고, 위치/배치만 비율에 따라 달라진다.
- 왼쪽 패널에 별도의 `공통 스타일` 섹션은 생기지 않는다.

## 제외 범위

- 이번 단계에서는 역할별 override UI를 별도로 노출하지 않는다.
- 카드 선택 강조나 세부 마이크로 인터랙션은 다루지 않는다.
- layout/contentArea의 z-order 규칙은 기존 합의대로 유지한다.
