  - 공식 템플릿의 카드 썸네일/레이아웃 이미지는 S3 public URL로 저장되어 DB에 들어간다.
  - 폰트는 아직 S3/DB로 일괄 이관되지 않았고, 렌더러는 로컬 파일 경로 또는 legacy fonts dir를 해석해서 읽는다.
  - `TemplateBuilderService`의 custom/system edit 저장은 아직 로컬 JSON repository 기반이다.
  - system template edit mode에서 수정한 뒤 `템플릿 등록`을 눌러야 official Postgres registry로 들어간다.
- 남은 문서화 포인트:
  - 나중에 폰트까지 포함해 asset 정책을 완전 DB/S3 통일할지 별도 설계가 필요하다.
  - custom template 저장소를 local JSON으로 둘지 DB로 이동할지 정책 결정을 다시 해야 한다.
